A **SuperPlane app** is a control plane for running long-lived, event-driven DevOps workflows. It works across the tools teams already use such as Git, CI/CD, incident response, observability, infra, notifications, and more.

An app includes a [canvas](/concepts/canvas) for the workflow, a [console](/concepts/console) for control and status, [memory](/concepts/canvas-memory) for application data, and a [git repository](/concepts/files) for its configuration.

## Getting started

**Create a blank app** — Empty canvas, console, memory, and repository; build out each part as you go.

**Install from the catalog** — Pre-built canvas, console, and memory; customize and publish like any other app.
